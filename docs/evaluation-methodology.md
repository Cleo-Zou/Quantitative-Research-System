# 指数增强基金研究解读框架设计

## Fund Research Interpretation — Rule-based Research Engine

> **定位**: 本框架定义"如何解读一只指数增强基金"。它不是评分模型，而是一套**可复现、可解释的研究规则体系（Rule-based Research Interpretation）**——将研究员的分析逻辑抽象为确定性规则，相同数据输入始终得到一致的研究结论。系统输出不是分数或投资建议，而是**基金画像（Profile）、研究标签（Tags）和研究摘要（Research Summary）**。

## 一：评价标准

### Mission

> **能够持续、稳定地创造超越基准指数的收益，同时在风险可控的情况下保持增强能力。**

所有后续规则围绕这句话展开

### Five Principles

#### Principle 1 — 同类比较

所有研究解读均仅在**相同 Benchmark 的增强基金之间**完成。沪深 300 增强只与沪深 300 增强比较，不跨基准比较。

#### Principle 2 — 超额优先

Alpha 代表基金经理的主动管理能力，是研究解读的核心维度。绝对收益受市场 Beta 影响，不能直接等同于管理能力。

#### Principle 3 — 持续性

持续、稳定地创造 Alpha 比单次爆发更有价值。研究关注各周期 Alpha 的方向一致性与趋势，而非仅看单一周期。

#### Principle 4 — 风险收益匹配

Alpha 必须建立在合理的风险水平之上。高 Alpha 伴随极端回撤，其能力可信度低于中等 Alpha 且风险可控的基金。

#### Principle 5 — 可解释性

每一项研究结论均应能够追溯至明确的数据来源与确定性规则，不依赖人工判断或黑盒模型。

## 第二章：评价边界

### Evaluation Scope

**适用对象**: 指数增强基金，即明确以超越标的指数为投资目标的公募基金产品。

**不适用于**:

* 主动权益基金（无明确 Benchmark 跟踪目标）

* 被动指数基金 / ETF / LOF（以复制而非增强为目标）

* QDII 基金（受汇率、跨境市场结构影响，不可直接比较）

* Smart Beta 产品（因子暴露逻辑不同于传统增强策略）

* 债券增强基金（风险收益特征与权益增强基金不具备可比性）

**重要声明**:

* 所有研究解读仅基于历史数据，不代表未来表现。

* 研究结论为基于确定规则自动生成的解读，不构成投资建议。

* 基金经理和投资策略可能发生变化，历史 Alpha 可能不再延续。

## 第三章：评价对象与评价框架

### 评价对象

研究解读的核心是**基金经理创造 Alpha 的能力**，而非基金的绝对收益。收益是结果，Alpha 才是能力来源。框架将两者分开分析。

### 五步解读框架

研究解读按以下固定顺序展开，每一步回答一个研究问题：

| 步骤              | 维度      | 核心问题                            |
| --------------- | ------- | ------------------------------- |
| ① Skill         | 主动管理能力  | 基金经理有没有创造 Alpha？                |
| ② Consistency   | 持续性     | 这个 Alpha 稳不稳定？是一贯的还是偶然的？有没有延续性？ |
| ③ Outcome       | 投资结果    | 投资者最终赚没赚到钱？                     |
| ④ Risk-adjusted | 风险调整后表现 | 承担了多少风险换来的 Alpha？               |
| ⑤ Summary       | 综合画像    | 这只基金整体是一幅什么画像？                  |

> **为什么 Skill 排在 Outcome 前面？**
> 研究员首先关心的是"基金经理有没有能力"，因为市场涨跌是临时因素，Alpha 能力才是长期因素。先判断能力，再结合市场环境理解结果，还原了真正的研究逻辑。

## 第四章：Profile 画像系统

将每只基金归类为一种**研究画像（Profile）Profile 描述基金的**长期 Alpha 特征，不随短期表现频繁切换。趋势变化通过 Tag 体现。

### 四种基金画像

| Profile    | 定义                  | 典型特征                         |
| ---------- | ------------------- | ---------------------------- |
| **稳定增强型**  | 持续创造 Alpha，主动管理能力稳定 | 各周期 Alpha > 0，Alpha 波动小，回撤可控 |
| **高弹性增强型** | Alpha 突出但波动较大       | 一年超额显著，但伴随较高回撤或 Alpha 波动     |
| **弱增强型**   | Alpha 接近零，增强特征不明显   | 各周期超额在 ±0.5% 附近              |
| **指数复制型**  | 长期未展现正 Alpha        | 各周期超额基本为负或长期为零               |

> **状态说明**: 成立不足一年的基金标记为 "观察样本"，不分配 Profile。

### Profile 判定逻辑

Profile 之间**互斥**，每只基金分配一个核心画像。Profile 描述基金的长期 Alpha 特征，不随短期波动频繁切换。趋势变化与短期表现通过 Tag 体现。

```
if 成立不足一年:
    → 观察样本（无 Profile）

elif 各周期 Alpha 全部 > 0:
    if 一年超额排名同类 top 20%:
        if 最大回撤明显高于同类（偏离中位数超过缓冲区）:
            → 高弹性增强型
        else:
            → 稳定增强型
    elif Alpha 波动小（各周期间最大差异 < TREND_TOLERANCE）:
        → 稳定增强型
    else:
        → 稳定增强型

elif 各周期 Alpha 全部 ≤ 0:
    → 指数复制型

elif 一年超额绝对值 < ALPHA_NEUTRAL 且 六月/一月绝对值 < ALPHA_NEUTRAL:
    → 弱增强型

else:
    → 弱增强型
```

> **注意**: "高弹性增强型"要求回撤**明显**高于同类（而非仅略高于中位数），避免中位数附近随机波动导致画像误判。具体缓冲区间见"参数校准"章节。

## 第五章：标签系统

标签分两层：**状态标签**（互斥，只有一个）和 **研究标签**（可多个叠加）。趋势变化通过标签体现，而非修改 Profile。

### 第一层：状态标签（Status）

| 标签   | 含义          | 触发条件     |
| ---- | ----------- | -------- |
| 观察样本 | 数据不足以参与完整研究 | 成立 < 1 年 |
| 正常   | 可参与完整研究解读   | 成立 ≥ 1 年 |

### 第二层：研究标签（Tags）

| 标签         | 含义                   | 触发条件                                                          |
| ---------- | -------------------- | ------------------------------------------------------------- |
| &#x20;持续跑赢 | 各周期均跑赢基准             | 一年/六月/一月超额全部 > 0                                              |
| Alpha 稳定   | 各周期 Alpha 差异极小，无明显趋势 | 一年/六月/一月超额两两差异 < TREND_TOLERANCE                              |
| 超额突出       | Alpha 排名同类前列         | 一年超额排名同类 top 10%（或绝对阈值，视样本量）                                  |
| 短期改善       | 主动管理能力在加速            | 一月 − 六月 > TREND_TOLERANCE 且 六月 − 一年 > TREND_TOLERANCE         |
| 近期回落       | 超额正在衰减               | 六月 − 一月 > TREND_TOLERANCE 且 一年 − 六月 > TREND_TOLERANCE，且一月 ≤ 0 |
| 低回撤        | 下行风险控制较好             | 最大回撤 < 同类中位数                                                  |
| 收益居前       | 绝对收益排名靠前             | 一年收益排名同类 top 30%（或绝对阈值，视样本量）                                  |
| 超额偏弱       | Alpha 接近零            | 一年超额绝对值 < ALPHA_NEUTRAL                                       |
| &#x20;超额落后 | Alpha 长期为负           | 一年超额为负且排名同类后 30%（或绝对阈值，视样本量）                                  |

> 一只基金可同时具有多个标签。例如 `持续跑赢 Alpha稳定 超额突出 低回撤` 就是"稳定增强型"的典型标签组合。
> `近期回落` 可以与 Profile "稳定增强型"共存——说明这只基金长期不错，但最近在掉队。这正是研究员需要关注的。

## 第六章：研究语言库

所有解读使用**相对表述**。每一项结论均可追溯至具体数据或排名。

### ① Skill（主动管理能力）语言库

| 场景     | 标准表述                              |
| ------ | --------------------------------- |
| 超额突出   | `近一年超额收益位于同类前 {N}%，体现出较强的主动管理能力。` |
| 超额居前   | `近一年超额收益位于同类前 {N}%，主动管理能力居前。`     |
| 超额居中   | `近一年超额收益处于同类中等水平，主动管理能力未体现明显优势。`  |
| 超额偏弱   | `暂未展现出明显的主动管理能力。`                 |
| 超额持续落后 | `各评价周期均未取得正超额，主动管理能力偏弱。`          |

### ② Consistency（持续性与延续性）语言库

| 场景        | 标准表述                                          |
| --------- | --------------------------------------------- |
| 持续为正 + 稳定 | `一年、六月及一月均跑赢基准，各周期超额差异较小，主动管理能力具有较好的持续性与稳定性。` |
| 持续为正      | `一年、六月及一月均跑赢基准，增强能力具有较好的持续性。`                 |
| 趋势改善      | `超额收益呈逐期改善趋势，近期超额表现具有一定延续性。`                  |
| 趋势平稳      | `各周期超额收益差异较小，主动管理能力保持平稳。`                     |
| 趋势回落      | `近期超额收益有所回落，主动管理能力的延续性有待观察。`                  |
| 明显衰减      | `一年超额尚可但近期持续走弱，主动管理能力出现衰减迹象。`                 |

### ③ Outcome（投资结果）语言库

| 场景   | 标准表述                                |
| ---- | ----------------------------------- |
| 收益居前 | `近一年收益位于同类前 {N}%，收益表现居前。`           |
| 收益居中 | `近一年收益处于同类中游水平。`                    |
| 市场拖累 | `受市场整体下跌影响，近一年绝对收益为负，但同期跑赢基准 {N}%。` |
| 收益偏弱 | `近一年绝对收益与超额均表现偏弱。`                  |

### ④ Risk-adjusted Performance（风险调整后表现）语言库

| 场景      | 标准表述                          |
| ------- | ----------------------------- |
| 回撤控制好   | `最大回撤优于同类 {N}% 的基金，下行风险控制较好。` |
| 回撤偏大    | `最大回撤高于同类多数基金，需关注下行风险。`       |
| 风险收益匹配好 | `在取得较好超额的同时回撤可控，风险收益匹配较好。`    |
| 风险收益待改善 | `超额收益突出但伴随较大回撤，风险收益匹配有待改善。`   |

> 后续扩展指标：Sharpe Ratio、波动率、Tracking Error、Information Ratio。

### ⑤ Summary（综合画像）语言库

| Profile | 标准结论                                   |
| ------- | -------------------------------------- |
| 稳定增强型   | `整体主动管理能力稳定，超额收益持续性较好，风险收益特征较优。`       |
| 高弹性增强型  | `主动管理能力突出，但波动/回撤偏高，超额收益的稳定性弱于同类稳定型基金。` |
| 弱增强型    | `主动管理能力尚不突出，超额收益与基准差异较小，增强特征有待进一步验证。`  |
| 指数复制型   | `未展现出显著的主动管理能力，收益表现与基准接近，增强效果不明显。`     |
| 观察样本    | `成立时间较短，历史数据不足以进行完整的研究解读，暂不纳入比较。`      |

## 第八章：参数校准

框架与参数分离。参数可随样本分布校准，不影响框架本身。

### 设计原则

| 原则      | 说明                                          |
| ------- | ------------------------------------------- |
| 框架与参数分离 | 评价哲学、五步框架、Profile/Tag 体系、语言库属于框架层，不因参数调整而修改 |
| 参数集中管理  | 所有阈值统一定义，修改一处即可全局生效                         |
| 样本自适应   | 阈值根据同类样本量自动选择百分位或绝对模式，避免小样本统计失真             |
| 缓冲区设计   | 分类边界预留缓冲区间，避免中位数附近微小差异导致画像/标签频繁切换           |

### 核心参数

#### 样本量阈值

| 参数                      | 建议值 | 说明                              |
| ----------------------- | --- | ------------------------------- |
| `MIN_SAMPLE_PERCENTILE` | 20  | 同类样本 ≥ 该值时采用百分位排名法；< 该值时改用绝对阈值法 |

> **原因**: 当同类基金少于 20 只时，top 10% 仅对应 2 只甚至 1 只基金，排名不具有统计意义。此时换用绝对 Alpha 阈值（如一年超额 > 3% 视为突出）更为稳健。

#### 排名百分位

| 参数              | 建议值  | 说明                            |
| --------------- | ---- | ----------------------------- |
| `TOP_EXCELLENT` | 0.10 | Alpha 突出的分位阈值（同类前 10%）        |
| `TOP_GOOD`      | 0.30 | Alpha 居前 / 收益居前的分位阈值（同类前 30%） |
| `BOTTOM_WEAK`   | 0.30 | Alpha 落后的分位阈值（同类后 30%）        |

> 仅在 `同类样本 ≥ MIN_SAMPLE_PERCENTILE` 时使用。样本不足时，所有百分位阈值替换为绝对阈值（由 `ALPHA_*_ABS` 系列参数定义）。

#### 绝对 Alpha 阈值（小样本模式下使用）

| 参数                    | 建议值   | 说明                |
| --------------------- | ----- | ----------------- |
| `ALPHA_EXCELLENT_ABS` | 3.0%  | 一年超额 > 该值视为突出     |
| `ALPHA_GOOD_ABS`      | 1.0%  | 一年超额 > 该值视为居前     |
| `ALPHA_NEUTRAL`       | 0.5%  | 一年超额绝对值 < 该值视为弱增强 |
| `ALPHA_WEAK_ABS`      | −0.5% | 一年超额 < 该值视为偏弱     |

#### 趋势容忍区间

| 参数                | 建议值  | 说明                                 |
| ----------------- | ---- | ---------------------------------- |
| `TREND_TOLERANCE` | 0.5% | 相邻周期 Alpha 差异 < 该值视为"平稳"，不识别为改善或回落 |

> **原因**: 一年 3.2%、六月 3.1%、一月 3.3% 这种情形并非真正的趋势，只是随机波动。趋势容忍区间确保只有在差异足够显著时才触发趋势判断。

趋势判断逻辑：

```
if abs(一年−六月) < TREND_TOLERANCE and abs(六月−一月) < TREND_TOLERANCE:
    → 趋势平稳
elif 一月 − 六月 > TREND_TOLERANCE and 六月 − 一年 > TREND_TOLERANCE:
    → 短期改善
elif 六月 − 一月 > TREND_TOLERANCE and 一年 − 六月 > TREND_TOLERANCE:
    → 近期回落
else:
    → 无明确趋势（不触发改善/回落标签）
```

#### Profile 分类缓冲区

| 参数           | 建议值            | 说明                       |
| ------------ | -------------- | ------------------------ |
| `MDD_BUFFER` | 0.5σ 或等价的百分位偏移 | 回撤需**明显**高于同类中位数才判定为高弹性型 |

> **原因**: 中位数附近 ±0.3% 的回撤差异可能是噪音。仅回撤显著偏高（如超过中位数 + 0.5σ 或处于同类后 30%）时归类为"高弹性增强型"，其余归入"稳定增强型"。避免因中位数附近微小差异导致画像误判。

### 参数校准流程

当样本分布变化（如新增基金、市场环境改变）时，可按以下步骤校准：

1. 运行 `05` 脚本，输出各 Benchmark 组的样本量及 Alpha 分布统计

2. 检查小样本组（< 20 只）的绝对 Alpha 阈值是否合理

3. 检查趋势容忍区间是否与 Alpha 的自然波动幅度匹配

4. 调整参数后重新运行，对比 Profile 分布变化

5. 参数稳定后锁定，纳入 `config.py`

## 附录：关键决策记录

| 决策                  | 理由                                                   |
| ------------------- | ---------------------------------------------------- |
|                     |                                                      |
| Profile 仅描述长期特征     | 趋势变化通过 Tag 体现，避免 Profile 频繁切换，更符合研究实践中"基金画像相对稳定"的认知。 |
| Skill 排在 Outcome 前面 | 研究员首先关心"基金经理有没有能力"，市场涨跌是临时因素，Alpha 才是长期因素。           |
| 全部用相对排名而非绝对数值       | "同类前 10%"比"超额 3%"有研究含义——前者建立了比较基准。                   |
| 不生成投资建议             | 研究解读描述"当前表现说明了什么"，买卖决策是基金经理的职责。                      |
| Rule-based 而非 AI 生成 | 确保可解释、可复现、可追溯——每条结论都可以反向拆解到具体规则和数据。                  |
| 全文使用"研究解读"而非"评价"    | 评价暗示好坏判断，研究解读意味着帮助理解数据。                              |
