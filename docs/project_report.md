
# 指数增强基金量化研究系统 — 项目报告

> **Quantitative Research System for Enhanced Index Funds**
>


---

## 摘要

本项目构建了一套完整的**指数增强基金量化研究系统**，覆盖基金池构建、净值采集、收益计算、风险指标评估、研究解读自动生成与可视化展示的全流程。系统以四大宽基指数（沪深300、中证500、中证1000、中证全指）的增强基金为研究对象，基于 **Rule-based Research Interpretation Engine（规则驱动的研究解读引擎）**，自动生成包含五核心指标、基金画像（Profile）、研究标签（Tags）和五段研究摘要的完整分析报告。系统部署于 GitHub Actions，实现每个交易日的全自动数据更新与发布。

## 第一章：项目概述

### 1.1 研究目标

指数增强基金的核心价值在于**持续、稳定地创造超越基准指数的超额收益（Alpha）**。本系统旨在回答以下研究问题：

- 哪些指数增强基金真正具有主动管理能力？
- 如何在同类基金中横向比较收益能力、风险控制和风险调整后表现？
- 如何将研究员的分析逻辑抽象为可复现、可解释的规则体系？

### 1.2 研究对象

| 基准指数 | 增强基金数量 | 代表产品示例 |

| 中证全指 (CSI_ALL) | 16 只 | 博道中证全指增强、国金中证全指增强 |

## 第三章：核心指标计算

### 3.1 区间涨跌幅

采用 App 展示口径（对齐天天基金/支付宝）：

日收益率与累计净值变化的关系：

$$r_t = \frac{NAV_t}{NAV_{t-1}} - 1$$

> **符号说明：**
> - $r_t$：第 $t$ 个交易日的收益率
> - $NAV_t$：第 $t$ 个交易日的基金净值
> - $NAV_{t-1}$：前一个交易日的基金净值

N 日累计收益率：

$$R_{cumulative} = \frac{NAV_{latest}}{NAV_{base}} - 1$$

> **符号说明：**
> - $R_{cumulative}$：区间累计收益率
> - $NAV_{latest}$：区间最新交易日的净值
> - $NAV_{base}$：区间起始日的净值

其中短期（日/周）使用**单位净值（unit_nav）**，中长期（月及以上）使用**复权净值（adj_nav）**以消除分红拆分影响。

### 3.2 超额收益（Alpha）

超额收益定义为基金收益与 95% 基准指数收益之差（指增基金仓位上限 95%，乘以 0.95 系数还原真实超额能力）：

$$\alpha_t = R_t^{fund} - 0.95 \times R_t^{benchmark}$$

> **符号说明：**
> - $\alpha_t$：第 $t$ 期的超额收益（Alpha）
> - $R_t^{fund}$：基金在第 $t$ 期的收益率
> - $R_t^{benchmark}$：基准指数在同期的收益率
> - $0.95$：仓位系数（指数增强基金股票仓位上限为 95%，约 5% 为现金/债券以应对申赎）

本系统进一步做**股息修正**，剥离指数分红对超额的贡献：

$$\alpha_t^{pure} = \alpha_t - \left[ (1 + DY)^{years} - 1 \right]$$

> **符号说明：**
> - $\alpha_t^{pure}$：股息修正后的纯 Alpha
> - $DY$：指数股息率（dividend yield），从中证指数公司获取
> - $years$：计算区间对应的年限（如近一年 = 1.0，近六月 = 0.5）修正后的纯 Alpha 更真实地反映基金经理的主动选股能力。

### 3.3 年化收益率（Annual Return）

基于全量复权净值的几何年化：

$$R_{annual} = \left( \frac{NAV_{end}}{NAV_{start}} \right)^{\frac{252}{N}} - 1$$

> **符号说明：**
> - $R_{annual}$：年化收益率
> - $NAV_{end}$：最新交易日的复权净值
> - $NAV_{start}$：历史起始日的复权净值
> - $N$：起始日至最新日之间的交易日总数
> - $252$：A 股市场一年大约的交易天数（年化因子）

### 3.4 年化波动率（Annual Volatility）

先计算日收益率序列：

$$r_t = \frac{NAV_t}{NAV_{t-1}} - 1, \quad t = 1, 2, ..., N$$

> **符号说明：**
> - $r_t$：第 $t$ 个交易日的收益率
> - $N$：交易日总数

日收益率标准差：

$$\sigma_{daily} = \sqrt{ \frac{1}{N-1} \sum_{t=1}^{N} (r_t - \bar{r})^2 }$$

> **符号说明：**
> - $\sigma_{daily}$：日收益率的标准差
> - $\bar{r}$：日收益率的均值，$\bar{r} = \frac{1}{N}\sum_{t=1}^{N} r_t$
> - $N-1$：自由度修正（样本标准差）

年化：

$$\sigma_{annual} = \sigma_{daily} \times \sqrt{252}$$

> **符号说明：**
> - $\sigma_{annual}$：年化波动率
> - $\sqrt{252}$：将日波动率放大到年（假设收益率独立同分布，方差随时间线性累加）

### 3.5 最大回撤（Maximum Drawdown）

定义历史最高净值序列：

$$Peak_t = \max_{1 \leq i \leq t} NAV_i$$

> **符号说明：**
> - $Peak_t$：从第 1 天到第 $t$ 天之间的最高净值
> - $NAV_i$：第 $i$ 个交易日的复权净值

时点回撤：

$$Drawdown_t = \frac{NAV_t - Peak_t}{Peak_t}$$

> **符号说明：**
> - $Drawdown_t$：第 $t$ 天的回撤幅度（负值 = 亏损比例）
> - $NAV_t$：第 $t$ 天当前的净值
> - $Peak_t$：第 $t$ 天之前的历史最高净值

最大回撤：

$$MDD = \min_{1 \leq t \leq N} Drawdown_t$$

> **符号说明：**
> - $MDD$：整个历史区间内回撤的最大值（绝对值最大的负值）
> - 例如 $MDD = -0.15$ 表示历史上最惨的时候净值从最高点下跌了 15%

### 3.6 夏普比率（Sharpe Ratio）

衡量每单位风险所获得的超额回报：

$$Sharpe = \frac{R_{annual} - R_f}{\sigma_{annual}}$$

> **符号说明：**
> - $Sharpe$：夏普比率（越高越好）
> - $R_{annual}$：基金的年化收益率
> - $R_f$：无风险利率，本系统取 $2\%$（近似一年期定存利率）
> - $\sigma_{annual}$：基金的年化波动率
>
> **含义**：如果 Sharpe = 0.8，意味着每承担 1% 的波动风险，获得 0.8% 的超额回报。

### 3.7 卡玛比率（Calmar Ratio）

衡量年化收益与最大回撤的关系：

$$Calmar = \frac{R_{annual}}{|MDD|}$$

> **符号说明：**
> - $Calmar$：卡玛比率（越高越好）
> - $R_{annual}$：基金的年化收益率
> - $|MDD|$：最大回撤的绝对值（取正数）
>
> **含义**：如果 Calmar = 2，意味着每承担 1% 的最大回撤，获得 2% 的年化收益。

## 第四章：同类比较与评价体系

> 五核心指标的具体计算公式见第三章。本章聚焦如何在同一 Benchmark 内横向比较基金。

### 4.1 同类排名方法

所有评价均在相同 Benchmark 的增强基金之间完成（Like-for-Like Comparison）。

**大样本模式**（同类 ≥ 20 只）：使用百分位排名法。

对于指标 $x$，组内排名（升序，值越大越好）：

$$rank_i = |\{j : x_j \leq x_i\}|$$

> **符号说明：**
> - $rank_i$：基金 $i$ 的组内排名（1 = 最好）
> - $x_i$：基金 $i$ 在该指标上的取值
> - $|\{...\}|$：集合中元素的个数（即有多少只基金的指标值 ≤ $x_i$）

百分位：

$$pct_i = \frac{rank_i}{N_{group}} \times 100\%$$

> **符号说明：**
> - $pct_i$：基金 $i$ 的百分位（越小越靠前，pct=1% 表示同类前 1%）
> - $N_{group}$：同基准下的基金总数

**小样本模式**（$N_{group} < 20$ 只）：改用绝对阈值法（如一年超额 > 3% 视为突出），避免小样本下百分位失真。

### 4.2 Alpha 三周期评估

| 周期 | 指标 | 用途 |
|------|------|------|
| 近一月 | month_1_alpha | 短期趋势判断 |
| 近六月 | month_6_alpha | 中期持续性评估 |
| 近一年 | year_1_alpha | 长期超额能力 |

趋势判断引入**容忍区间** $\epsilon = 0.5\%$（`TREND_TOLERANCE`），避免微小波动被误判为趋势：

$$\text{趋势} = \begin{cases} \text{改善} & \alpha_{1m} - \alpha_{6m} > \epsilon \;\land\; \alpha_{6m} - \alpha_{1y} > \epsilon \\ \text{回落} & \alpha_{6m} - \alpha_{1m} > \epsilon \;\land\; \alpha_{1y} - \alpha_{6m} > \epsilon \\ \text{平稳} & \text{otherwise} \end{cases}$$

> **符号说明：**
> - $\alpha_{1y}$：近一年超额收益（`year_1_alpha`）
> - $\alpha_{6m}$：近六月超额收益（`month_6_alpha`）
> - $\alpha_{1m}$：近一月超额收益（`month_1_alpha`）
> - $\epsilon$：趋势容忍区间，默认 0.5%，相邻周期 Alpha 差异小于此值视为随机波动
>
> **含义**：只有当相邻周期的 Alpha 变化幅度超过 $\epsilon$ 时，才认定为真正的趋势变化，否则视为平稳。

## 第五章：研究解读引擎

### 5.1 设计哲学（Five Principles）

1. **同类比较原则（Like-for-Like）**：所有评价仅在相同 Benchmark 基金之间完成
2. **超额优先原则（Alpha First）**：Alpha 代表主动管理能力，是评价核心维度
3. **持续性原则（Consistency over Burst）**：持续创造 Alpha 比单次爆发更有价值
4. **风险收益匹配原则（Risk-Adjusted）**：Alpha 必须建立在合理风险水平之上
5. **可解释性原则（Explainability）**：每条结论可追溯至明确数据与规则

### 5.2 五步解读框架

| 步骤 | 维度 | 核心问题 |
|------|------|----------|
| ① Return | 收益能力 | 基金赚了多少钱？有没有跑赢基准？ |
| ② Risk | 风险控制 | 波动多大？最惨亏多少？ |
| ③ Risk-Adjusted | 风险调整收益 | 每单位风险换来多少回报？ |
| ④ Consistency | 超额持续性 | Alpha 是稳定的还是一时运气？ |
| ⑤ Summary | 综合画像 | 这只基金整体是什么画像？ |

### 5.3 基金画像（Profile）分类

| Profile | 判定条件 | 解释 |
|---------|----------|------|
| 稳健增强型 | Alpha > 0 且 Sharpe/Calmar 居前，回撤可控 | 超额与风控均衡 |
| 风险收益优化型 | Sharpe + Calmar 双高 + 低波动（Alpha 可能不突出） | 风险调整收益优秀 |
| 高弹性增强型 | 收益/Alpha 突出但波动或回撤显著偏高 | 进攻型选手 |
| 普通增强型 | 各项指标处于同类中游 | 增强能力待验证 |
| 指数复制型 | 各周期 Alpha 均 ≤ 0 | 未展现主动管理能力 |

### 5.4 研究标签（Tags）体系

标签分两层：

**状态标签（互斥）**：
- `[观察样本]` — 成立不足一年，数据不足以参与完整评价
- `正常` — 可参与完整研究解读

**能力标签（可叠加）**：
- `[持续跑赢]` — 各周期 Alpha 均 > 0
- `[超额突出]` — 一年超额排名同类前 10%，且 Alpha > 0
- `[Alpha稳定]` — 各周期间 Alpha 差异 < 容忍区间
- `[短期改善]` — Alpha 呈逐期加速趋势
- `[近期回落]` — Alpha 呈逐期衰减趋势
- `[高收益]` — 年化收益排名同类前 30%
- `[低波动]` — 年化波动率低于同类 70%
- `[高Sharpe]` — Sharpe Ratio 排名同类前 30%
- `[高Calmar]` — Calmar Ratio 排名同类前 30%
- `[低回撤]` — 最大回撤低于同类 70%

## 第七章：项目亮点

1. **Rule-based Research Interpretation**：不使用大模型，将研究员的分析逻辑抽象为可复现、可解释的规则体系。相同数据输入始终得到一致结论。

2. **五核心指标体系**：覆盖收益（Annual Return）、风险（Volatility、Max Drawdown）和风险调整收益（Sharpe、Calmar）三个维度，超越单一指标评价。

3. **Profile + Tags + Summary 三层研究输出**：不采用难以解释的五星评分，而是画像、标签、研究摘要的组合，更贴近内部研究报告的表达方式。

4. **参数与框架分离**：评价哲学（Methodology）固定，参数（Parameters）可根据样本分布校准，支持小样本自适应切换。

5. **全自动 CI/CD**：代码推送到 GitHub → 每日自动更新 → 自动部署到网页，无需人工干预。
